Optimize - Getting Started Guide Document

The Webtrends Optimize Mobile SDK is a library for iOS that enables your iOS application to implement multivariate and A/B testing. Elements within your application can be changed according to experiments you set up under the Webtrends Optimize product. Once it is installed and configured, you may create Optimize project to test factors in the UI elements or within the behavior of the code of your app.

Functional Overview

There are two basic roles that the Optimize™ SDK plays in your application:
1. Change the look and feel of UI elements
2. Track the effectiveness of those changes by tracking conversion rates

For more inforamtion about tracking conversions, see our Tracking Conversions Guide.

Creating an app that uses the Optimize™ SDK is simple. Just follow these basic steps:

  1. Create an app.
  2. Install the SDK.
  3. Configure some of your UI elements to be Optimize™-enabled.
  4. Create a project in Optimize that can act on those elements.
  5. Run the application and see the effects.

Installation

To install the SDK into your application, follow the installation instructions in the Getting Started Guide. Below is a walkthrough of the entire process of getting the SDK integrated into your application.

An Example Application

Once your project has the Optimize SDK installed, you may begin creating UI elements and code that will respond to Optimize tests. For this example, we will create a simple application that contains two Optimize-enabled UI elements, an image and a button.

  1. Create a new project. For this example, we are creating a single view controller and using the Swift language to build the application.
  2. Once the project is created, close Xcode.
  3. Open a terminal window and navigate to the directory of the new project.
  4. Install Webtrends SDK via Cocoapods.
  5. Create a webtrends.plist file.
  6. Create a bridging header (if you are using the Swift programming language).
  7. In the storyboard, create a UIImageView and a UIButton on the main viewController. Change the class of these elements to WTOptimizeImageView and WTOptimizeButton.
  8. Give these UI elements a wtIdentifier value. The wtIdentifier can be set in the attributes inspector panel. This identifier will be used as a target within Optimize to apply UI changes. For this example, I have called the image myOptImage and the button myOptButton.
  9. For this page, we will be creating a project in the Optimize UI. This project must point to a location, which must be set in our webtrends.plist file using the wt_opt_project_locations key. I have called my project location simpleOpt.app/mainPage. For help on using the Optimize™ product, see the online help documentation.
  10. In Optimize, create a location as a New Web Page. Name the location anything you wish and set the Web Page Address as simpleOpt.app/mainPage. No other information is required to create the location.
  11. After the location is created, you must create a site that points to the new location. For this example, create a site named simpleOptApplication and set the primary domain to simpleOpt.app.
  12. In Optimize, create a new project for this app. Name the project and set it as a Factorial project type. Set the location to the location you just created.
  13. Add two factors named myOptImage and myOptButton. Do not check the Script Only checkbox.
  14. For this project, we will skip creating a conversion event.
  15. Create a new test. Set the test state to Staging and All Visitors.
  16. For the images, create two levels, each with a different image URL in the script. Make sure the image factors are redirects.

  17. For the button, create two levels, each with a different JSON object such as:

    {
  "myOptButton": {
    "buttonText":{
      "normal":"Hit it!"
    },
    "backgroundColor":"#99ccff"
  }
}
  18. When you run the app, the SDK reads from the Optimize server, downloads the tests, downloads the images, and stores them locally. In this simple app, there is not enough time to do all that overhead work before the first (and only) view is displayed. However, if the application is terminated and restarted, the new content will become visible and will persist for future app launches.

For more information on using the Optimize™ product to create projects, tests, factors, etc., please see the Optimize Help Files.

Important Note: In this very simple app, the only view controller loads very quickly, which does not allow the SDK time enough to ask the server for tests, download the tests, then download the replacement image. As a result, when the app is launched, you will not see any of the optimization tests displayed. However, if the app is closed and reopened, the optimizations will be displayed correctly.

This is not a bug and can be prevented by avoiding the display of optimizations on a launch screen. By the time a user navigates to subsequent views in the app, the content should be displayed. For this reason, it is always a good idea to have sane default values for all Optimize UI elements such as images and text. In the event that a user launches the application without any network connectivity, your defaults will be displayed and there will be no negative impact to the user experience. This will only affect the experience the first time the app is launched. Any subsequent launches will pull data from a local cache and will be nearly instantaneous, whether the network is available or not.

Installing the Webtrends SDK Pods

  1. Once the project is created, close Xcode.
  2. Open a terminal window and navigate to the directory of the new project.

  3. Create a Podfile file in this directory containing:

     pod 'Webtrends-SDK/Core'
 pod 'Webtrends-SDK/Optimize'
  4. Run the pod install command from the terminal window.
  5. Once the pods are successfully installed, open the .xcworkspace file. Do not open the old .xcodeproj file.

Create a webtrends.plist File

Create a Property List file by selecting File -> New -> File… and choosing Property List from the Resource group. Name the file webtrends.plist.

The webtrends.plist file contains required and optional configuration settings. The only required entries in this file are:

For Analytics™:

  • wt_dc_dcsid: Your Webtrends Analytics™ DCSID for this application

For Optimize™:

  • wt_opt_key_token: Your Webtrends Optimize™ key token
  • wt_opt_domain_id: Your Optimnize™ domain ID, also known as an account number
  • wt_opt_account_guid: Your Webtrends Account GUID. This is a 10-character alphanumeric string. Do not include the “.acct.webtrends.com” suffix.
  • wt_opt_project_locations: A comma-separated list of project locations. For this example, it will contain the project location that you define above in the Optimize™ application.
  • wt_opt_staging_mode: Set this to true if you have created Optimize™ tests in staging mode. This is commonly used while developing applications and Optimize™ tests. Once the application is ready for launch, remove this setting or set it to false and change the tests in Optimize™ to Active.

Here is what your finished webtrends.plist file will look like. Your values will change, of course, but the keys will be the same as below.

Other configuration settings are optional and are detailed in the Analytics Advanced Configuration Guide or the Optimize Advanced Configuration Guide.

Create a Bridging Header (Swift apps only)

iOS apps written in Swift require a bridging header to read Objective-C libraries. Creating one is very simple.

  1. Select File -> New -> File… and choose Header File from the iOS Source group.
  2. Open the file and import the WebtrendsSDK.h and (optionally) the OptimizeSDK.h header files. Your file will look something like this:
    // You'll need to import WebtrendsSDK.h into your bridging header in order to use the SDK
    // from Swift. Apple's documentation has more information on how to set this up:
    // https://developer.apple.com/library/prerelease/ios/documentation/Swift/Conceptual/BuildingCocoaApps/MixandMatch.html

    @import Foundation;

    #import "WebtrendsSDK.h"
    #import "OptimizeSDK.h"
