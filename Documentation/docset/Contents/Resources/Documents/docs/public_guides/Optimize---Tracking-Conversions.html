Optimize - Tracking Conversions Document

About Conversion Events

Once your Optimize™ tests are set up on the server and in your application, you will want to track the results. This is done with Conversion Events. Conversion events are desired actions on a view controller in your application that a user performs. You can add conversion events to baseline, split, and multivariate projects.

In an iOS application context, there are only two conversion event types to manage:

  • Page View Events
  • Custom Conversion Events

Page view events should always precede conversion events. This enables the conversion event to be put into its context and allows Optimize™ to determine the context of the conversion, giving you data about the success rates of your Optimize™ tests.

Triggering a Page View Event

Use the [WTOptimizeManager triggerPageView:withTestAlias:projectLocation:] method to manually trigger a page view event. This is usually done in the viewDidLoad: method of the view controller. You must know the test alias and the project location that is associated with the conversion. A Page View event must be triggered prior to a conversion event. For more information on conversion events, see the online help files for Optimize™.

For example, it would be appropriate to call the method from within your viewDidLoad: delegate method:

override func viewDidLoad() {
   let manager = WTOptimizeManager.sharedManager()
   manager.triggerPageView(self, withTestAlias: "ta_WT", projectLocation: "WTBankSample.app/persistent")
}
- (void)viewDidLoad {
    [super viewDidLoad];
    // Do any additional setup after loading the view.
    WTOptimizeManager *manager = [WTOptimizeManager sharedManager];
    [manager triggerPageView:self withTestAlias:@"ta_WT" projectLocation:@"WTBankSample.app/persistent"];
}

For an example of triggering a Page View event, see our Sample Applications Guide and our Sample Applications Repo.

Triggering a Conversion Event

Once a page view event has been triggered, a conversion event may take place. A conversion event can be triggered in any way you choose in your application, such as a swipe, a button-tap, or any other user-driven action. Just put the [WTOptimizeManager triggerEventForConversionWithTestAlias:projectLocation:conversionPoint:] method into the handler for the specific action you wish to convert on. For example, consider the following snippets:

@IBAction func sendConversion(sender: AnyObject) {
        let manager = WTOptimizeManager.sharedManager()
        manager.triggerEventForConversionWithTestAlias("ta_WT", projectLocation: "WTBankSample.app/persistent", conversionPoint: "cp_clickedconvertbutton")
    }
- (IBAction)sendConversion:(id)sender {
    WTOptimizeManager *manager = [WTOptimizeManager sharedManager];
    [manager triggerEventForConversionWithTestAlias:@"ta_WT" projectLocation:@"WTBankSample.app/persistent" conversionPoint:@"cp_clickedConvertButton"];
}

The sendConversion: method could be the target action of a button or gesture in the current view controller.
