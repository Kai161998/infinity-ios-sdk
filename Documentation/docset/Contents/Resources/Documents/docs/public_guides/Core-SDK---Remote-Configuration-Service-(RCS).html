Remote Configuration Service

The Webtrends Mobile SDK supports the use of the Webtrends Remote Configuration Service (RCS). RCS enables developers to change configuration settings in your application after the application has been deployed. When RCS is enabled on the client application (by setting the wt_dc_rcs_enabled setting to ‘true’), the application will call home on a periodic schedule to check if there are any updates. If an update is received, the new config settings are applied to the application immediately.

Each RCS rule contains two elements:
1) A set of rules that tell the RCS server which clients to apply the change to, and
2) A set of configuration settings to set on the affected clients.

When the client phones home to the RCS server, it sends along a data set that describes the environment that the application is running in. For example, each request contains the device information (model, OS version, build number, screen resolution, carrier, language, country, etc.) and information about the application (name, version). The RCS server then checks to see if there is a rule defined for that configuration and responds by setting the defined configuration settings.

For example, you could create a rule that sets the Only Send Over WiFi to true for all AT&T Wireless customers in Canada. Any set of device/application parameters can be combined into a pseudo WHERE clause that can be used to apply a set of one or more configuration settings.

Contact Webtrends Support for more details on setting up specific rules for your application.
